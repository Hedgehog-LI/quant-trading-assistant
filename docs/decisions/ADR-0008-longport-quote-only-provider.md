# ADR-0008: LongPort 只作为只读行情 Provider

## 状态

Accepted

## 日期

2026-07-10

## 背景

LongPort/长桥 OpenAPI 和 MCP 同时覆盖行情、账户、资产组合和交易能力。当前项目需要接入 A 股行情用于数据存档、复盘和异常观察，但项目定位明确不是自动交易系统。

若直接把 LongPort 当“券商接入”引入，后续 AI 或代码生成工具可能误接下单、撤单、账户资金、真实持仓等能力，突破项目边界。

## 决策

本项目只允许把 LongPort 作为 `marketdata` 模块的只读行情 provider：

- 允许：基础信息、实时/准实时行情、历史 K 线、交易日历、数据质量检查。
- 禁止：下单、改单、撤单、账户资金、真实持仓、订单、成交、保证金、最大可买数量。

实现上：

- 后端只初始化 Quote 相关 client/provider。
- LongPort 凭据只从服务端环境变量或安全配置读取。
- 前端不输入、不保存、不展示任何 LongPort 密钥。
- 日志不得打印 token、secret、完整原始响应。
- CI 和单测使用 Fake provider，不依赖真实 LongPort 凭据。
- 未配置 provider 时系统仍可启动，返回 `configured=false`。

## 原因

- 保持本项目“交易辅助、复盘、数据沉淀”的定位。
- 降低 AI 开发误接真实交易能力的风险。
- 让行情数据和手工交易记录、手工估值、持仓快照保持清晰边界。
- 确保后续指标、策略、回测建立在可追溯的数据源上。

## 影响

- `marketdata` 模块会新增 provider 抽象、外部价格快照、同步任务和异常提醒。
- `portfolio_price_snapshot` 继续表示用户手工当前价，不会被 LongPort 行情覆盖。
- 若未来用户明确要求账户/持仓只读查询，也必须新增 ADR 和风控设计后再讨论，不默认开放。

## 替代方案

1. 直接使用 LongPort MCP 作为系统数据源：拒绝。MCP 更适合 AI 临时查询，不适合作为后端生产数据落库边界。
2. 接入完整 LongPort OpenAPI：拒绝。交易和账户能力超出项目定位。
3. 仅继续 CSV 导入：保留作为 fallback，但无法满足实时/准实时观察和自动补档需求。
