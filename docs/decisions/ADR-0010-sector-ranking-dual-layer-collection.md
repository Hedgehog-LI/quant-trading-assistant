# ADR-0010: 板块全榜与关注明细采用双层自动采集

## 状态

Accepted

## 背景

P1.5 只保存用户关注板块的手工快照。仅关注驱动会产生选择偏差，无法长期回答每天全市场领涨和领跌板块；对全部行业高频保存成分明细又会导致请求与存储无边界增长。

## 决策

1. 建立全市场排行批次，按市场一次抓取完整行业榜单，保存收盘与可选盘中快照。
2. 只有用户关注并显式开启自动采集的板块保存聚合和成分明细。
3. 盘中频率限定为 `5/10/15/30/60分钟`，`0` 表示仅收盘。
4. `enabled` 与 `auto_collect_enabled` 分离，历史关注升级后不自动发起请求。
5. 自动数据使用市场时间桶幂等，并用 DB claim 防止多实例重复执行。
6. 排行与关注明细共享 Provider 抽象和错误分类，但使用独立调度器，不侵入股票分钟 K scheduler。

## 后果

### 正面

- 低成本形成无选择偏差的全市场板块历史。
- 关注板块仍保留足够细的资金与成分解释能力。
- 可基于同一批次开发领涨/领跌、排名迁移和板块轮动。
- 自动任务具备幂等、退避和权限降噪。

### 代价

- 新增排行配置、批次、明细和自动调度状态。
- 多市场需要时区、交易时段与日历治理。
- 行业分类跨 Provider 不可直接合并，后续需要映射层。

## 不采用的方案

- **只采关注板块**：无法发现未提前关注的领涨行业。
- **全市场全部成分高频采集**：请求和成分快照行数增长过快。
- **复用股票分钟 K 表**：板块快照是截面聚合，不是可交易证券 OHLCV。
